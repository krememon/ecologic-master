Copy still fails on iOS native (“Failed to copy to clipboard”). This means we’re either:

NOT actually using Capacitor Clipboard at runtime (falling back to navigator.clipboard / execCommand)

Clipboard plugin isn’t installed/synced into iOS

The native build hasn’t been rebuilt after adding the plugin

Or the copy handler is still doing input-session operations (focus/select)

We need a definitive fix with proper native clipboard usage + verification.

Step 1 — Verify plugin is installed and synced

Check package.json for @capacitor/clipboard. If missing:

npm i @capacitor/clipboard

Run:

npx cap sync ios

Confirm iOS project has the plugin integrated (Pods/Package).

If you added the plugin, you MUST ensure the iOS wrapper was rebuilt/synced.

Step 2 — Implement a single source of truth helper: copyToClipboard(text)

Create src/utils/clipboard.ts (or similar):

Requirements:

Detect native via Capacitor:

import { Capacitor } from '@capacitor/core'

const isNative = Capacitor.isNativePlatform()

If native:

import { Clipboard } from '@capacitor/clipboard'

await Clipboard.write({ string: text })

return { ok: true }

If web:

try navigator.clipboard.writeText(text) if available

else fallback to hidden textarea + document.execCommand('copy')
Return { ok: boolean, method: 'capacitor'|'navigator'|'textarea', error?: string }

Important:

Never call .focus() or .select() on any visible input in the native path.

Do not require any active text input session.

Step 3 — Replace ALL copy usages

Search for:

“copy to clipboard”

navigator.clipboard

execCommand('copy')

“company code”
Replace every instance with the helper.
