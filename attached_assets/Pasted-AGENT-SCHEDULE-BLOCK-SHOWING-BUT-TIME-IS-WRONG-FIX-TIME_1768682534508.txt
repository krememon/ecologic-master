but the block is being placed at the WRONG time compared to what I selected in the schedule modal.

Example: I scheduled it for one time, but it renders at 2:00 PM (wrong).

Make schedule blocks render at the EXACT time the user selected (no hour offsets, no timezone drift).

IMPORTANT:

- Fix the time calculation and normalization only.

LIKELY ROOT CAUSE:
We are mixing:
- scheduledDate stored as a Date with UTC time component
- scheduledTime stored as "HH:mm"
and then converting between UTC/local incorrectly (double-applying time or shifting timezones).

In the schedule items normalization (server or client), log ONE estimate item (temporary):

The user-selected time MUST come from scheduledTime ("HH:mm").
Do NOT take hours/minutes from scheduledDate at all.

So the correct computation is:

A) Extract local date-only string from scheduledDate:
- dateOnly = YYYY-MM-DD in LOCAL timezone for the selected day

B) Combine dateOnly + scheduledTime into a LOCAL datetime (no UTC conversion):
- startAtLocal = new Date(`${dateOnly}T${scheduledTime}:00`)

- minutesFromMidnight = startAtLocal.getHours() * 60 + startAtLocal.getMinutes()

DO NOT:
- use Date.UTC
- use toISOString() for placement math
- parse scheduledDate’s time portion
- add timezone offsets manually
