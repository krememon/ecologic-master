MASTER CONTEXT (EcoLogic)
We are seeing Google OAuth fail ONLY inside the iPhone simulator wrapper with:
“Access blocked… Use secure browsers policy… Error 403: disallowed_useragent”.
On Replit (normal browser) Google login works. We confirmed we use Google OAuth 2.0 via `passport-google-oauth20`.

ROOT CAUSE
Google blocks OAuth flows inside embedded browsers / WKWebView. Our wrapper runs the `/auth/google` flow inside the WebView, so Google rejects it. Fix is to run Google auth in a “secure system browser” (Safari / ASWebAuthenticationSession) and then deep-link back into the app.

GOAL
Keep existing web OAuth flow unchanged.
For iOS wrapper only:
1) Open Google auth in system browser (Capacitor Browser)
2) After Passport callback succeeds, redirect to deep link: `ecologic://auth/callback?token=...`
3) App catches deep link, closes browser, stores token, and navigates to Dashboard.

IMPORTANT
- Do NOT attempt to do Google OAuth inside the WebView.
- Must still work on Replit/web exactly like before.

TASKS (BACKEND - Express + passport-google-oauth20)
1) Update the “start Google auth” route to accept `?platform=ios` and preserve it through the OAuth round-trip using the OAuth `state` param.
   - Example: GET `/auth/google?platform=ios`
   - Use passport authenticate with `state: "ios"` when platform=ios, else state:"web" or omit.

2) Update the Google callback route `/auth/google/callback`:
   - After successful passport auth, generate our normal EcoLogic session token (JWT preferred).
   - Read platform from `req.query.state` (Google returns it back).
   - If platform === "ios":
        redirect to `ecologic://auth/callback?token=<ENCODED_TOKEN>`
     else:
        keep existing behavior (redirect to web app route like `${APP_URL}/auth/callback?token=...` or whatever we currently do).

3) If we currently only use cookies/sessions for web:
   - Implement a short-lived JWT token for the wrapper redirect OR implement a “one-time code” exchange endpoint:
       - callback redirects to `ecologic://auth/callback?code=...`
       - app calls `/api/auth/exchange-code` to get session token.
   - Choose the simplest approach that matches existing auth architecture; do not break web login.
