1) Make sure the thread is the same for both users

If Owner and Supervisor landed on different threads, inserts go to A and the other listens to B.

// threadService.ts
export function makeDeterministicThreadKey(a: string, b: string) {
  return [a, b].sort().join('__'); // e.g., "ownerId__supervisorId"
}

export async function getOrCreateThread(aId: string, bId: string) {
  const key = makeDeterministicThreadKey(aId, bId);
  const ref = doc(db, 'threads', key);
  const snap = await getDoc(ref);
  if (!snap.exists()) {
    await setDoc(ref, {
      members: [aId, bId],
      createdAt: serverTimestamp(),
      updatedAt: serverTimestamp(),
    });
  }
  return key; // use as threadId everywhere
}


Use this when you navigate to the chat screen so both users share the same threadId.

2) Ensure Firestore rules were deployed

If the rules weren’t published, creates will silently fail in UI.

Rules should be exactly:

rules_version = '2';
service cloud.firestore {
  match /databases/{db}/documents {
    match /threads/{threadId} {
      allow read: if request.auth != null && request.auth.uid in resource.data.members;
      match /messages/{messageId} {
        allow read, create: if request.auth != null
          && request.auth.uid in get(/databases/$(db)/documents/threads/$(threadId)).data.members;
      }
    }
  }
}


After editing, Deploy them in the Firebase console (or whatever your deploy step is).

3) Bubble up send errors (so we see what’s wrong)

Wrap your send with a visible toast/log.

const handleSend = async (text: string) => {
  const tempId = crypto.randomUUID();
  const optimistic = { id: tempId, text, senderId: meId, createdAt: new Date() };
  setMessages((cur) => [...cur, optimistic]);

  try {
    await addDoc(collection(db, 'threads', threadId, 'messages'), {
      text,
      senderId: meId,
      createdAt: serverTimestamp(),
    });
  } catch (err) {
    console.error('sendMessage failed:', err);
    setMessages((cur) => cur.filter(m => m.id !== tempId));
    alert('Message failed to send. Check console for details.');
  }
};


If you see a permission error, it’s the rules or members array. If you see not-initialized, it’s Firebase config.

4) Confirm the realtime listener is mounted on the right path

Double-check threadId in the subscribe call matches the one you insert into.

useEffect(() => {
  if (!threadId) return;
  const q = query(
    collection(db, 'threads', threadId, 'messages'),
    orderBy('createdAt', 'asc')
  );
  const unsub = onSnapshot(q, (snap) => {
    const msgs = snap.docs.map(d => ({ id: d.id, ...d.data() })) as Msg[];
    setMessages(msgs);
  });
  return () => unsub();
}, [threadId]);
