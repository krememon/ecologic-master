Fix Price Book multi-select bug: allow adding MULTIPLE line items (all prices + totals must reflect every selected item)

CURRENT
- When creating a Job and adding line items from the Price Book:
  - I can click/select multiple items
  - BUT only ONE item’s price/amount actually shows up in the job/invoice totals
- This breaks billing accuracy and is a blocker for real use

GOAL
Price Book line item picker must support true multi-select:
- If user selects 2+ items, ALL selected items are added as separate line items
- Each line item includes correct:
  - description/name
  - qty (default 1)
  - unit price
  - line total
- Overall totals must equal the sum of ALL selected items
- Works the same everywhere we use the Price Book picker:
  - Job creation
  - Estimate creation
  - Invoice creation (including tech invoice flow)
No fake data. No UI redesign.
