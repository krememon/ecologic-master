Fix Payments “Pending” amount to show unpaid money (Accounts Receivable), not $0.00

CURRENT
- Payments page Breakdown shows:
  Cash / Check / Card / Pending
- Cash/Check/Card represent paid money
- Pending currently shows $0.00 and is not tied to what customers still owe

GOAL
“Pending” must represent the total amount of money owed but NOT yet paid:
- Pending = total unpaid invoice balances
- Updates instantly when payments are recorded (cash/check/card/stripe)
- Must be accurate and auditable

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
DEFINITIONS (CRITICAL)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Paid totals:
- Cash/Check/Card totals = sum of completed payments by method

Pending total:
- Pending = SUM( invoice_total_cents - amount_paid_cents ) for invoices where balance > 0
- This includes:
  - Unpaid invoices
  - Partially paid invoices (if supported)
- Excludes:
  - Paid invoices (balance = 0)
  - Cancelled/void invoices (if supported)

If you do not currently store amount_paid_cents:
- Compute it from payments table:
  amount_paid_cents = SUM(payments.amountCents) for that invoice where status = "paid"

Use real data only:
- invoices table (invoice totals + status + jobId/companyId)

1) Add a backend endpoint for breakdown totals:
GET /api/payments/breakdown?range=this_month|last_month|custom&start=&end=
Returns:
- cashTotalCents
- checkTotalCents
- cardTotalCents
- pendingTotalCents
- completedTotalCents (optional)
- countCompleted (optional)

2) Calculation rules:
- cash/check/card totals:
  SUM(payments.amountCents) where method matches and status="paid" within date range
- pending total:
  For invoices in the selected range (same range):
    invoiceBalanceCents = invoiceTotalCents - SUM(paid payments for that invoice)
  pendingTotalCents = SUM(max(invoiceBalanceCents, 0)) across invoices where balance > 0

3) Range consistency:
- If range is “This Month”, both:
  - Completed payments total
  - Pending invoice balances
  must reflect invoices/payments in that same range definition

4) Update UI to use cents formatting:
- Convert cents → dollars
- Keep existing breakdown UI
