Build the first real feature for the private EcoLogic dashboard in STAGING only.

Feature:
Campaign/source tracking from dashboard tracking links into onboarding and subscriber attribution.

Current working foundation:
- staging-dashboard.ecologicc.com loads the private dashboard
- dashboard login works
- admin APIs are under /api/admin/dashboard/*
- existing tables include:
  - growth_subscribers
  - growth_campaigns
  - growth_creators
- dashboard access is controlled by DASHBOARD_ADMIN_EMAILS
- staging.ecologicc.com still loads the normal customer app

Important:
Do not touch production.
Do not break existing customer app routing, login, onboarding, Stripe, Apple IAP, Google Play IAP, jobs, billing, or subscriptions.
Build and test this in staging only.

Goal flow:
1. I create a campaign in the private dashboard.
2. Dashboard generates a tracking link.
3. Someone opens the tracking link on staging.ecologicc.com.
4. EcoLogic captures source/ref from the URL.
5. Onboarding pre-fills referral/source fields.
6. When onboarding/company setup finishes, EcoLogic saves the subscriber attribution.
7. The dashboard Subscribers page shows who came from that campaign/source.

PART 1 — Campaigns page UI

Build out the dashboard Campaigns page.

Admins should be able to:

- View all campaigns
- Create a campaign
- Edit a campaign
- Deactivate/reactivate a campaign
- Copy tracking link

Campaign fields:

- name
- sourceType
- sourceName
- referralCode
- trackingUrl
- cost
- status
- notes

Source type options:

- instagram_creator
- tiktok_creator
- supply_house
- flyer
- customer_referral
- cold_call
- google
- app_store
- organic
- other

Rules:
- referralCode is required
- referralCode should be normalized lowercase in the DB
- display referralCode uppercase in the UI if helpful
- referralCode must be unique case-insensitively
- status should be active/inactive
- cost can be blank or 0
- notes optional

Tracking link format for staging:

https://staging.ecologicc.com/signup?source=SOURCE_TYPE&ref=REFERRAL_CODE

Example:

https://staging.ecologicc.com/signup?source=instagram_creator&ref=joeplumbing

Later production should use:

https://app.ecologicc.com/signup?source=SOURCE_TYPE&ref=REFERRAL_CODE

Generate the tracking link based on environment/hostname. For staging dashboard, use staging.ecologicc.com.

PART 3 — Client attribution capture utility

In the normal customer app, create a small attribution utility that runs early on app load.

It should detect URL params:

- ref
- source
- campaign

Example:

https://staging.ecologicc.com/signup?source=instagram_creator&ref=joeplumbing

When detected, save attribution to localStorage and cookie if possible.

Saved data shape:

{
  referralCode: "joeplumbing",
  sourceType: "instagram_creator",
  campaignParam: null,
  firstSeenAt: ISO timestamp,
  lastSeenAt: ISO timestamp,
  landingPath: current path
}

Rules:
- first-touch attribution wins
- if attribution already exists with referralCode/sourceType, do not overwrite it
- update lastSeenAt only
- keep attribution for 90 days
- normalize referralCode lowercase
- normalize sourceType lowercase
- run this early in the customer app load
- do not run this on dashboard hostnames

Add safe logs:

[attribution] captured from URL
[attribution] existing attribution kept
[attribution] no attribution params found

PART 5 — Backend attribution save

When onboarding completes or company is created:

- read submitted source answer
- read submitted referral code
- also accept saved attribution values sent from the client if available
- normalize referral code lowercase
- try to match referralCode against growth_campaigns.referralCode
- only match active campaigns

If a matching campaign exists:
- create or update growth_subscribers for this user/company
- set:
  - userId
  - companyId
  - ownerEmail
  - companyName
  - sourceType
  - sourceName
  - referralCode
  - campaignId
  - signupAt if known
  - onboardingCompletedAt
  - platform = unknown for now
  - subscriptionStatus = unknown or trialing depending on current app logic

If no campaign match exists:
- still create/update growth_subscribers if the user provided source/referral info
- campaignId can be null
- sourceName can come from the selected source answer or be blank

Important:
- Do not overwrite existing attribution once set.
- One company/user should keep its original source unless manually changed later.
- Do not fail onboarding if saving attribution fails. Log it and continue.

Add logs:

[attribution] onboarding attribution received
[attribution] matched campaign
[attribution] no campaign match
[attribution] saved subscriber attribution
[attribution] attribution already exists, keeping original
[attribution] failed to save attribution but onboarding continued
