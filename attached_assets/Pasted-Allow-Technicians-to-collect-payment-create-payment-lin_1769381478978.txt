Allow Technicians to collect payment (create payment links) ONLY for invoices on jobs they are assigned to + remove loud 403 UI

CURRENT
- Technician can reach the Checkout/Pay screen
- Checkout shows error:
  403: "You do not have permission to create payment links"
- Tech must be able to take payment on-site for assigned jobs
- Current UI shows a loud error banner (not silent success)

GOAL
Technicians can collect payment ONLY for invoices tied to jobs they are assigned to:
- Cash / Check works
- Card works (Stripe payment link / checkout creation)
- After successful payment, job.paymentStatus flips Pending → Paid immediately everywhere (no refresh)
- RBAC is enforced server-side (assigned jobs only)
- No loud red error banners for expected behavior (hidden-not-disabled + silent)

- Allowed to create payment links ONLY when:
  - Invoice belongs to a job the tech is assigned to

- Keep existing payment behavior unchanged

2) Update permission logic:
- If role is Owner/Admin/Supervisor/Dispatcher: unchanged

  - Verify invoice is unpaid

  - Else -> return 403

  - payment.createdByRole = "Technician"
  - timestamp

- If payment link already exists for that invoice and is still valid:
  - Reuse it (do not spam Stripe with duplicates)
