Goal

When a user’s credentials are correct but they don’t belong to a company, login must succeed and the app must redirect to /join-company (not show an error).

1) Login endpoint: don’t error on “no company”

After verifying credentials:

Fetch membership: companyId (or row in CompanyUser).

Do not throw if missing. Create session normally and set a flag on the session payload, e.g.:

{ "ok": true, "needsCompany": true, "companyId": null }


If the account is actually wrong (bad password), still return 401 as usual.

If you use JWT/NextAuth-style callbacks, include companyId (nullable) and needsCompany in the token/session.

2) Client sign-in handler: route based on the flag

On successful response:

If needsCompany === true → router.replace('/join-company')

Else → router.replace('/home') (or your landing route)

Important: Only show the red “Something went wrong” on network/500/401.
Map any custom 409/428 from server to the join flow, not an error toast.

3) App middleware/route guard (server & client)

If authenticated and companyId == null:

Allow only: /join-company, /logout, /api/companies/join, static assets.

Redirect any other app route → /join-company.

If authenticated and companyId != null and the user visits /join-company:

302 redirect → /home to prevent the 404/flicker.

4) Join Company API stays the source of truth

POST /api/companies/join validates the code and links the user to that company.

On success: return { ok: true, companyId }.

Client then router.replace('/home') and invalidates session/user queries.

5) Error message mapping (no false “wrong”)

Login form should not show the generic “Something went wrong” when the server returns “needs company.” That’s a success path.

Only show the red error for:

401 bad credentials

500 server error

Everything else → route logic above.

6) OAuth too

Apply the same logic in your OAuth callback:

If email matches an account with no company, set needsCompany: true and route to /join-company.

7) Acceptance checks

Remove a user from company → try to sign in:

✅ Login succeeds (no error banner).

✅ Immediately lands on /join-company.

Enter a valid invite code:

✅ Linked to company and redirected to Home.

Enter an invalid code:

✅ Inline red error “Invalid invite code.”

Already-in-company user visiting /join-company:

✅ Redirected to Home (no 404).

Bad password:

✅ Stays on login with “Invalid email or password.”
