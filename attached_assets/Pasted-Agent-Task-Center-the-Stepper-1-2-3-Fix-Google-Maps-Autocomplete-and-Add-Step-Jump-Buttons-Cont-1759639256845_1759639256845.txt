We need three improvements:

Center the stepper bubbles (1-2-3) at the top of the modal.

Fix Google Maps / Places Autocomplete so the Location field works.

Add UI buttons so PJ can directly jump to Step 2 or Step 3 for testing.

Ensure the Location input uses Google Places Autocomplete and is functional.

Implementation Requirements (high level—no inline code)
A) Center the Stepper

In the wizard header container:

Use a flex column layout.

First line: title “Create New Job”.

Second line: the stepper centered (use a single flex row with equal gaps).

Third line: a small caption that displays the current step label (“Job Details”, “Client”, “Schedule”).

The three bubbles must visually indicate state:

Active (filled), Done (muted fill), Upcoming (outline).

Keep Back / Next / Finish buttons in the footer; they remain functional.

B) Add Step Jump Buttons (QA shortcuts)

Add two small text buttons visible on Step 1 only:

“Go to Client (Step 2)”

“Go to Schedule (Step 3)”

These set the current step directly without changing validation state.

Keep normal Next behavior gated by validation (title/location, etc.).

Add an inline note: “Dev shortcuts (for testing)”.

C) Fix Google Maps / Places Autocomplete

Libraries & Loader

Use @react-google-maps/api loader with libraries=['places'].

Ensure a single loader instance ID (e.g., id: 'ecologic-places-loader') to avoid duplicate script loads.

Environment Key

Read VITE_GOOGLE_MAPS_KEY from env. Document this in the README snippet below.

Enable APIs & Billing

In Google Cloud console, enable Maps JavaScript API and Places API for the same project as the key.

Billing must be enabled.

HTTP Referrer Restrictions (common failure point)

Add these referrers:

http://localhost:*/* (for Vite dev)

https://*.replit.dev/* (Replit preview)

(Optional) Your production domain(s) if deployed.

Wire Autocomplete to Location input

Replace the raw text input for Location with a Places Autocomplete wrapper.

On place_changed read formatted_address and update the location state.

Do not block typing while Google loads; show a “Loading Google Places…” placeholder if needed.

Graceful Fallback

If the loader errors or the API key is missing, fall back to a normal text input so the user can still proceed.

City/ZIP Extraction (optional for now)

If easy, parse address_components to auto-fill City and ZIP. Otherwise leave for a follow-up task.

Common Google Maps Errors & How to Resolve

InvalidKeyMapError: Key typo or wrong project. Verify key string and project.

RefererNotAllowedMapError: Add correct referrers (localhost, *.replit.dev) to the key restrictions.

ApiNotActivatedMapError: Enable Maps JavaScript API and Places API for the key’s project.

BillingNotEnabledMapError: Attach a billing account in Google Cloud.
