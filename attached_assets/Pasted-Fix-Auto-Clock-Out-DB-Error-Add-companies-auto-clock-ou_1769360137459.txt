3) Restore /api/auth/user to always return 200 for valid sessions

Backend code (storage.getUserCompany / auth user endpoint) is selecting a column that doesn’t exist in the companies table.

Add the column to the companies table:

- Column name: auto_clock_out_time
- Type: TEXT (or TIME if you already use TIME types)

Also ensure existing rows get the default.

Example SQL (choose one approach based on your DB tooling):
1) If TIME type supported and desired:
   ALTER TABLE companies

2) If using TEXT:
   ALTER TABLE companies
   ADD COLUMN IF NOT EXISTS auto_clock_out_time TEXT NOT NULL DEFAULT '18:00';
