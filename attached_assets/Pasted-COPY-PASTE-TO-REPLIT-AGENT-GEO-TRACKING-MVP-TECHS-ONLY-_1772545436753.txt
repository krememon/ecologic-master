TRACKING CONFIG (Slightly Optimized “Google Maps feel”)

Distance filter: send a point when moved ≥ 50 meters

Heartbeat: send a point every 60 seconds even if not moving

Accuracy filter: ignore very bad GPS points (e.g., accuracy > ~100m) so markers don’t jump

Batch sending: buffer points locally and POST in batches every 30–60 seconds (or immediately if buffer reaches N points)

2) API routes

POST /api/time/clock-in

Creates session row

Returns sessionId

MUST succeed even if location permission is denied.

POST /api/time/clock-out

Closes active session

POST /api/location/batch

Body: { sessionId, points: [{lat,lng,accuracy,recordedAt,source}] }

Validate:

user authenticated

sessionId belongs to user and is active (clock_out_at is null)

org matches

Insert points

Return ok

GET /api/location/active
Return latest location per “active” user (clocked in) filtered by RBAC.

Response items:
{ userId, name, avatarUrl, lat, lng, lastUpdatedAt }

Only include users whose lastUpdatedAt is within 10 minutes (server-side stale filter)

RBAC enforcement (server-side, mandatory):

If role == Owner/Admin: include all active users in org

If role == Supervisor: include only active users tied to jobs the supervisor can access (use your existing job access logic)

If role == Technician: only include self
