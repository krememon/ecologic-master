0) Clarify “Signature”

In Stripe, the “signature/confirm” step is usually:

3D Secure / bank authorization (only appears for some cards)

OR Stripe Checkout confirmation screen

We cannot force 3DS for every card, but we can test it using a 3DS test card and ensure checkout is configured correctly. The key is: whatever Stripe shows, once it finishes, we must auto-return to the app.

1) Root cause of “Safari can’t redirect”

This happens when:

Stripe redirects to a custom scheme (ecologic://...) from a browser context that blocks it, OR

The return URL is a replit.dev page trying to JS-redirect and iOS blocks it, OR

We’re using a tap here page instead of a real app-return mechanism.

Fix: Use Universal Links (https) as the return URL + have the app intercept the link and close the Browser. This is the most reliable iOS behavior.

https://ecologicc.com (preferred)

https://app.ecologicc.com

B) Stripe success/cancel URLs
