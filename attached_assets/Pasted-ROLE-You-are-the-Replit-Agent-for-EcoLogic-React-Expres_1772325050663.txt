C) Route: Create PaymentIntent for an invoice
POST /api/payments/stripe/create-intent
Body: { invoiceId }
Auth: required. RBAC: allow Owner/Admin + Dispatcher + Supervisor (if they have job access) + Estimator if allowed by your business rules. Technicians only if you explicitly want them to collect payment (default: NO).
Server:
1. Load invoice by invoiceId, validate user can access it.
2. Compute amount due in cents (invoice total - already paid).
3. Create a Stripe PaymentIntent:
   - amount, currency
   - automatic_payment_methods: { enabled: true } (or keep to card only if you want)
   - metadata: { invoiceId, ecologicPaymentLedger: 'true' }
4. Upsert a payments ledger row in DB with:
   - method='card', provider='stripe'
   - status='requires_payment_method' (or 'processing')
   - provider_ref = paymentIntent.id
5. Return: { clientSecret, publishableKey, paymentIntentId, amountCents, currency }

D) Webhook: Stripe -> EcoLogic (source of truth)
POST /api/webhooks/stripe
- MUST use raw body middleware ONLY on this route to verify signature.
- Verify signature with STRIPE_WEBHOOK_SECRET.
Handle events:
1) payment_intent.succeeded
   - Find payment row by provider_ref = pi.id (or by metadata.invoiceId fallback).
   - Idempotency: if payment.status already 'succeeded' return 200.
   - Update payment.status='succeeded'
   - Optionally fetch latest charge to store receipt_url, brand/last4, charge_id.
   - Update invoice amount_paid/status to paid/partial.
   - Trigger QuickBooks sync (async job or background queue; if none exists, call existing sync function but guard with quickbooks_payment_id idempotency).
2) payment_intent.payment_failed
   - Update payment.status='failed'
3) charge.refunded (or refund.updated)
   - Create a refund record OR set payment.status='refunded' (choose simplest: set refunded if full refund; partial refund can be added later).
Return 200 quickly.

E) Route: Get Stripe publishable key
GET /api/payments/stripe/config -> { publishableKey }
