Build an Accounts management section inside the private EcoLogic dashboard in STAGING only.

Context:
We now have a private admin dashboard at staging-dashboard.ecologicc.com.
It already tracks campaigns, subscribers, sources, and Stripe subscription sync.
I now want to manage EcoLogic customer accounts from this private admin dashboard.

Important:
Build in STAGING only.
Do not touch production.
Do not break the customer app, login, billing, Stripe, Apple IAP, Google Play IAP, jobs, onboarding, or subscriptions.
All new APIs must stay under the admin dashboard namespace:
/api/admin/dashboard/*

Dashboard access must remain controlled by DASHBOARD_ADMIN_EMAILS only.
Do not grant access based on normal customer company role.

Goal:
Add an “Accounts” page to the private dashboard where approved admins can view and manage customer companies/accounts.

PART 4 — Safe account management actions

For V1, add only safe actions first:

1. Edit attribution/source
Allow admin to manually update:
- sourceType
- sourceName
- campaignId
- referralCode

2. Edit admin notes
Allow internal notes on the account.

3. Mark account status
Allow:
- active
- inactive
- test
- internal
- blocked

This should be an internal dashboard status only unless an existing app-level status already exists.

4. Refresh subscription data
Button:
Refresh Subscription

This should re-check current known Stripe subscription/customer data if available and update the dashboard fields.

Do NOT add destructive actions yet:
- Do not delete companies
- Do not delete users
- Do not permanently cancel subscriptions unless we explicitly add it later
- Do not impersonate users yet

PART 5 — Backend APIs

Create protected admin-only APIs:

GET /api/admin/dashboard/accounts
GET /api/admin/dashboard/accounts/:companyId
PATCH /api/admin/dashboard/accounts/:companyId/attribution
PATCH /api/admin/dashboard/accounts/:companyId/status
PATCH /api/admin/dashboard/accounts/:companyId/notes
POST /api/admin/dashboard/accounts/:companyId/refresh-subscription

All routes must be dashboard-admin-only.

PART 6 — Data source

The Accounts page should join data from:
- companies table
- users/memberships table
- growth_subscribers table if it exists
- growth_campaigns table if attribution exists
- subscription/billing/company fields already used by EcoLogic
- Stripe customer/subscription IDs if available

Important:
If a company has no growth_subscribers row yet, still show it in Accounts.
If an admin manually adds attribution to an account, create or update a growth_subscribers row for that company.
