Make remote push notifications work end-to-end (backend → APNs → iPhone) using the new APNS_ secrets*, while guaranteeing Apple Sign-In is untouched.

NON-NEGOTIABLE SAFETY RULES

DO NOT rename, delete, or modify any existing Apple Sign-In secrets (APPLE_TEAM_ID, APPLE_KEY_ID, APPLE_PRIVATE_KEY, etc.).

DO NOT change any Apple Sign-In routes, auth flows, tokens, callbacks, polling, or UI.

Push notifications must use ONLY:

APNS_TEAM_ID

APNS_KEY_ID

APNS_PRIVATE_KEY

Topic / bundle id: com.ecologic.app (or APNS_TOPIC override)

If you need a Team ID, use APNS_TEAM_ID only (even if it matches APPLE_TEAM_ID).

REQUIRED DELIVERABLES

POST /api/push/register (auth required) saves the device token for the logged-in user (already exists; verify it still works).

POST /api/push/test (auth required) sends a real APNs push to the saved token:

title: EcoLogic

body: Remote push works ✅

sound: default

API returns JSON with sent, failed, and failures details.

Console logs show APNs provider initialized and APNs result summary.

Apple Sign-In still works exactly the same (no regressions).

process.env.APNS_TEAM_ID

process.env.APNS_KEY_ID

process.env.APNS_PRIVATE_KEY

Topic must be process.env.APNS_TOPIC || "com.ecologic.app".

production should be process.env.NODE_ENV === "production".

Add clear logs:
