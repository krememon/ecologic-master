GOAL:
When an invoice becomes PAID (payment successful), send a push to OWNER user(s).

Search the backend for where invoice status is set to “paid”, “PAID”, “succeeded”, “SUCCESS”, or similar.

- "payment_intent" / "stripe" / "webhook"

We must send the notification from the place that ACTUALLY changes the invoice status, not from the UI button.

STEP 2 — ADD HARD LOGS AT THE STATUS CHANGE
At the exact line where invoice status changes, add:

If you can’t easily know fromStatus, fetch current invoice first and log it.

STEP 3 — TRIGGER NOTIFICATION ONLY ON TRANSITION TO PAID
Only send push when:
fromStatus !== "PAID" AND toStatus === "PAID"

Then call notification service:
type: "invoice_paid"
title: "Payment Received"
body: `Invoice paid – $${amount}`
data: { type:"invoice_paid", entityType:"invoice", entityId: invoiceId, linkUrl:`/invoices/${invoiceId}` }

STEP 4 — RESOLVE RECIPIENTS = OWNER ONLY

getCompanyOwners(companyId) -> userIds[]

Add logs:
