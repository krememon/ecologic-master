Right now the flow is:
Stripe → redirects to /stripe/return → shows “Returning…” inside the Stripe browser → stays there (because browser is still open) → user is stuck.

REQUIRED FLOW

Pay → confirmation → redirect → browser closes automatically → wrapper app is visible again → app refreshes subscription → route to Jobs.

1) Fix the mismatch: ecologicc.com vs app.ecologicc.com

Your Xcode Associated Domains says:

applinks:app.ecologicc.com

But backend says return URLs prioritize APP_BASE_URL (ecologicc.com).

That means Universal Links may NOT fire because the domain doesn’t match.

Do this:

Choose ONE domain for universal links (recommended: app.ecologicc.com)

Stripe success/cancel must use THAT exact domain:

https://app.ecologicc.com/stripe/return?...

AASA must be served from that exact domain:

https://app.ecologicc.com/.well-known/apple-app-site-association

Associated Domains must match:

applinks:app.ecologicc.com

If we keep ecologicc.com, then Associated Domains must be applinks:ecologicc.com. No mixing.

2) Stop relying on “page renders in main WKWebView”

This is the core misunderstanding. /stripe/return is loading in the Stripe browser, not the main WKWebView. Storing flags in localStorage doesn’t help because it’s the browser’s storage context, not the wrapper’s.

We need a mechanism that causes iOS to switch from the browser back to the app.

3) Correct universal-link return strategy (works without manual close)

We must make Stripe return to a universal link domain, then have the app intercept it via appUrlOpen and immediately close Browser.

A) In native app: add App.addListener('appUrlOpen')

If url path begins with /stripe/return:

await Browser.close()

parse query params (result + session_id)

call /api/subscriptions/status and update state

route success → /jobs, cancel → /billing

B) In the browser return page /stripe/return, do NOT “redirect to /”

For universal links, the browser is supposed to hand off to the app. The return page should be minimal and not try to “help” by redirecting or changing pages.
