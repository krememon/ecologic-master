Fix QuickBooks OAuth connection not persisting (still shows “Connect QuickBooks” after redirect)

CURRENT
- Clicking “Connect QuickBooks” redirects to Intuit and then back to EcoLogic
- After returning, Settings still shows “Connect QuickBooks” (not connected)
- Clicking again repeats the redirect loop
- This means tokens/realmId are not being saved OR UI is not reading saved state

GOAL
After a successful Intuit OAuth connection:
- EcoLogic must store realmId + tokens to the correct company in DB
- Settings must immediately show “Connected to QuickBooks”
- No manual refresh required
- No redirect loop

Implement a reliable “state” strategy:

On /connect:
- Generate a state value that encodes companyId + userId securely
  Option A (preferred): store state -> {companyId,userId} in DB table (qbo_oauth_states) with expiry (10 min)
  Option B: signed token (HMAC) that includes companyId/userId and expires quickly

- Resolve companyId/userId from state

- Only Owner/Admin can initiate connect
