Fix EcoLogic chat screen layout so it always fits fully inside the phone viewport without needing page scroll to see the top back arrow OR the bottom message input bar.

Current issue: chat renders at desktop-scale inside a phone-sized frame. The container isn’t constrained to a true mobile viewport height, so header + messages + input don’t lock correctly.

REQUIREMENTS
- The chat page must be a strict vertical flex layout:
  Header (fixed)
  Messages list (flex:1, scrollable)
  Input bar (fixed at bottom, always visible)
- The overall chat container must be constrained to the viewport height using mobile-safe units: 100dvh (fallback to 100vh if needed).
- Prevent the entire page from scrolling; only the messages list should scroll.
- Add safe-area padding for iOS: env(safe-area-inset-top/bottom)

DO THIS:

1) GLOBAL HEIGHT CONSTRAINTS
In global CSS (index.css / globals.css / App.css), ensure:
html, body, #root { height: 100%; }
body { margin: 0; overflow: hidden; }  // stop page scroll on chat layout

In the Chat page component (the route that renders the phone chat screen, likely Messages page / Conversation page), wrap the whole chat UI with a root div:

- height: 100dvh (or Tailwind h-dvh / min-h-dvh)
- display:flex; flex-direction:column;
- overflow:hidden;
- padding-top: env(safe-area-inset-top)
- padding-bottom: env(safe-area-inset-bottom)

Example (Tailwind):
<div className="h-dvh flex flex-col overflow-hidden pt-[env(safe-area-inset-top)] pb-[env(safe-area-inset-bottom)]">

If Tailwind doesn’t support h-dvh, use inline style:
style={{ height: '100dvh' }}

3) HEADER FIXED
Header should be:
flex: 0 0 auto;
height: (e.g. 56px)
