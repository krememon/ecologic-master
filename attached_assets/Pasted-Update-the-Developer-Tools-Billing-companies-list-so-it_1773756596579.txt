- hide blocked companies from the default list
- hide inactive/paused companies from the default list unless they still count as active access in your billing model

Use the same effective billing/access resolver already created, so the definition of “active” is consistent.

- active

- all

But for now:
- default UI should only show active

Update the header count so it reflects only active companies, not total companies in the database.

Example:

instead of total all-company count

5) ROW STATUS
For the active-only list:
- keep status badges if useful
- but rows in this default view should all be active/allowed companies

or equivalent default server-side filtering

Make sure filtering uses the actual effective access logic, not just a random raw field.

7) IMPORTANT

- Do not break company selection
