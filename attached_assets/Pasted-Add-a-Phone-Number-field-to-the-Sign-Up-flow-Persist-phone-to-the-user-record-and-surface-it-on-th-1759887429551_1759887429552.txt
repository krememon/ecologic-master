Add a Phone Number field to the Sign Up flow.

Persist phone to the user record and surface it on the Employees → Employee Information panel.

Show + edit phone on the Profile card inside Settings for all roles.

Keep RBAC as-is (no new permissions).

Data & API

Schema

Add phone (string, nullable) to the user/employee table (whatever backs employees).

Create an index for faster lookups if you already index email; phone is not unique.

Validation / Normalization

Accept digits, spaces, dashes, parentheses, plus sign, and x for extensions.

Normalize to E.164 on save when possible (e.g., +16315551234).

Store raw input if E.164 parse fails but still allow saving.

Client-side format-as-you-type (mask) for US numbers; don’t hardcode US—allow international.

API

Include phone in:

Auth/registration endpoint (create path).

User update endpoint used by Profile.

Employee read endpoints (list + detail).

Add server-side validation mirror (length 7–20 chars after stripping non-digits except leading + and x).
