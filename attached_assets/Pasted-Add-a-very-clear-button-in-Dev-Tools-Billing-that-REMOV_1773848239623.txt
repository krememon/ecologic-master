Add a very clear button in Dev Tools Billing that REMOVES the company’s subscription access.

GOAL
I need one obvious admin button that takes away the company’s subscription access so they stop being treated as subscribed.

IMPORTANT
This is dev-only for my allowlisted account only.
Use plain English.
Do not expose this to normal users.
Require confirmation before applying it.

WHAT THIS BUTTON SHOULD DO
Create a button called:

- Remove Subscription Access

Meaning:
- remove/clear the company’s active subscription entitlement
- remove any manual billing access that is making them count as subscribed, if that is the action I choose
- force the company to re-evaluate against the real billing resolver
- if they have no remaining valid access source, they should become blocked and hit the paywall

VERY IMPORTANT
I need this to be clear and separated into the right actions, because “subscription access” can mean different things.

BUILD IT AS 2 CLEAR OPTIONS:

1) REMOVE PAID SUBSCRIPTION STATUS
Button:
- Remove Paid Subscription Status

Use this when I want to strip the company of being treated like it has an active paid plan.

This should:
- clear any internal/manual plan override that is making them look paid
- clear subscription-related manual access if appropriate
- refresh billing state
- re-run the real resolver

If they still have:
- free access
- bypass
- active trial
then they can still remain allowed

2) FULLY REMOVE BILLING ACCESS
Button:
- Remove All Billing Access

This is the stronger button.

This should:
- turn off subscription bypass
- turn off free access
- remove manual plan override
- remove manual user limit override if tied to billing override logic
- leave real Stripe/trial truth alone unless you are intentionally simulating removal
- re-run the billing resolver immediately

If after recalculation there is:
- no real active Stripe subscription
- no active trial
- no free access
- no bypass
then:
- company becomes blocked
- company should go to paywall on next access check
