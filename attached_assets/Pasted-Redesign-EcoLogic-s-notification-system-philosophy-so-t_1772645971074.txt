Redesign EcoLogic’s notification system philosophy so the notification bell stops feeling like a spam folder. Right now, too many system events are being treated as bell notifications, which makes the bell feel noisy, cluttered, and low-value. I want the bell to feel like a smart attention center, not a dumping ground for every event.

Core goal

Refactor the notification experience so the bell primarily surfaces:

items that need user attention

important updates the user would be upset to miss

clear, actionable alerts

Lower-value operational events should either be:

deprioritized visually

grouped differently

filtered out of the main bell experience

or moved into a quieter “activity/update” style section if that can be done cleanly without overbuilding

New notification philosophy

The bell should not behave like a system log.

Instead, structure notifications conceptually into 3 buckets:

1) Action Required

These should be highest priority and feel most prominent in the bell.
Examples:

direct messages

job assigned / unassigned

job cancelled

major schedule/reschedule affecting today

estimate approved

invoice overdue

payment failed

missed clock-out

anything requiring manager/owner action

2) Important Updates

Still useful, but less urgent than true action items.
Examples:

payment collected

refund issued

payment succeeded

technician clocked in/out (for managers, if kept)

certain significant status changes

3) Activity / Low Priority

These should not dominate the bell and should feel quieter if they remain visible.
Examples:

generic job status changes

repetitive reminders like job_starting_soon

announcements

minor system updates

duplicate/overlapping payment confirmations

Use the existing notification infrastructure, but improve how notifications are classified and displayed.

Possible implementation approaches (choose the cleanest one):

add a backend priority/category field if that is cleaner

add a render grouping layer in the panel

default-sort by priority first, then recency

optionally collapse or visually soften low-priority notifications
