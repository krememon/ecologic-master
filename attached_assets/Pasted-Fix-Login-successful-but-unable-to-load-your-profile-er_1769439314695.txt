Fix “Login successful but unable to load your profile” error (auth/profile fetch failing after login)

CURRENT
- After signing in, UI shows:
  “Login successful but unable to load your profile. Please try refreshing the page.”
- This blocks access to the app and QuickBooks settings
- This appears immediately after login (session/profile fetch failure)

GOAL
Make login reliable:
- After successful login, the app MUST load the user profile every time
- No red error box for normal login
- No manual refresh required
- Fix the real failing request (401/403/500) and the root cause

- GET /api/session

Record:
- endpoint
- status code

STEP 2 — FIX ROOT CAUSE (MOST LIKELY)

Fix whichever is true:

A) Cookie/session not being sent
- Ensure login response sets cookie correctly
- Ensure cookie options are correct for replit.dev domain:
  - SameSite and Secure settings must allow cookie on HTTPS
- Ensure frontend fetch includes credentials:
  - fetch(..., { credentials: "include" })
  - or axios with withCredentials=true
- Ensure CORS allows credentials if applicable

B) Auth middleware mismatch
- Login sets one auth mechanism, profile endpoint expects another
- Make them consistent

C) Profile endpoint error due to missing company/user join
- If profile endpoint expects companyId and it’s null for this user, it may crash
- Handle “no company yet” gracefully and still return user profile

D) Redirect timing race
- App redirects before cookie/session is persisted
- Add a short, silent retry for profile fetch (max 2 retries) before showing any error

IMPORTANT:
- Do not show a loud red error for expected states
- If truly unauthenticated, redirect back to login silently
