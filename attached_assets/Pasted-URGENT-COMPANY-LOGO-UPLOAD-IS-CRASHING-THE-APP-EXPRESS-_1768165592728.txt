- Crash stack trace shows Express router/index.js lines (process_params / next / handle)
- Node v20.x
This means our upload endpoint or middleware is throwing an uncaught error.

GOAL
- Uploading a PNG/JPG logo (<=5MB) must NEVER crash the server.
- If upload fails, return a clean JSON error + keep server running.
- After success: store file, save company.logoUrl, UI shows preview immediately, persists after refresh.

DO THIS IN ORDER

1) FIND THE EXACT CRASHING ROUTE
- Locate the route that is hit when uploading the logo (ex: POST /api/company/logo or similar).

- Add TEMP logs right inside that route BEFORE any middleware runs:

2) FIX THE MOST COMMON MULTER/UPLOAD CRASHES
- Ensure we use multer correctly:
  - The frontend sends FormData with field name "file"
  - Backend MUST use upload.single("file")
- If current backend uses a different field name, align them (prefer "file").

- Ensure upload destination exists (if local storage):

Add this at the end of backend app setup AFTER routes:

    console.error("[SERVER ERROR]", err);
    if (res.headersSent) return next(err);
    res.status(err.status || 500).json({ error: err.message || "Server error" });
  });

Wrap the upload middleware so Multer errors return JSON instead of crashing:

5) AFTER CRASH IS FIXED, COMPLETE THE FEATURE
