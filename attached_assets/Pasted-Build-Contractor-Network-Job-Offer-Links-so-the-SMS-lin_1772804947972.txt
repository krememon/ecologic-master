Build “Contractor Network Job Offer Links” so the SMS link opens INSIDE the EcoLogic app (not a public job page).

Desired behavior:
- Sender taps “Send Job Offer” in EcoLogic → referral is created as usual.
- EcoLogic sends SMS to contractor phone with a link.
- If recipient HAS EcoLogic installed → link opens the EcoLogic app directly to a “Job Offer” screen with full job details + Accept / Decline.
- If recipient does NOT have EcoLogic installed → link sends them to the App Store / Google Play to download EcoLogic.
- After install + first open + login, EcoLogic should continue the flow and land them back on the Job Offer screen using the same token.

We are using Capacitor (iOS + Android). Implement Universal Links / App Links + in-app deep link handling.

========================================================
A) URL + TOKEN FORMAT (single link works everywhere)
========================================================
Use a single HTTPS link in the SMS:

https://app.ecologicc.com/invite/referral/<token>

- token is a secure random string stored on job_referrals
- This URL must be a Universal Link (iOS) + App Link (Android)

========================================================
B) DATABASE: job_referrals invite fields
========================================================
Add fields to job_referrals if missing:
- invite_token (text unique not null)
- invite_sent_to_phone (text nullable)
- invite_sent_at (timestamp nullable)
- invite_expires_at (timestamp nullable, default now + 7 days)

When POST /api/referrals/send creates a referral:
- generate invite_token
- set invite_expires_at
- set invite_sent_to_phone based on selected contractor’s phone
- set invite_sent_at = now

========================================================
C) SMS CONTENT (Telnyx later, queue now if blocked)

========================================================
We must configure:
1) iOS Universal Links
2) Android App Links

Domain to use: app.ecologicc.com (preferred) OR current prod domain if different.
(Do NOT rely on replit.dev for long-term universal links; use your real domain.)

========================================================
Universal/App Links will open browser if app not installed.
We need that web URL to redirect users to the appropriate app store.

Implement a very small landing handler for GET /invite/referral/:token on the web server:
- It should NOT show job details.
- It should show “Get EcoLogic” with buttons:
  - “Download on the App Store”
  - “Get it on Google Play”
- Attempt to open the app via deep link scheme as a secondary fallback if desired (optional).
- Include the token as a parameter so after install/open it can resume.

This page is only for “app not installed” fallback, NOT the offer details.

========================================================
F) IN-APP DEEP LINK HANDLING
========================================================
Implement deep link routing in the Capacitor app:

- When the app opens via URL https://app.ecologicc.com/invite/referral/:token
  route to:
  /referrals/invite/:token  (a new in-app screen)

If user is NOT logged in:
- show login/signup
- after login, continue automatically to /referrals/invite/:token

If user IS logged in:
- fetch invite details via API (below)
- show Job Offer screen with job details + Accept / Decline

========================================================
G) BACKEND API FOR INVITE DETAILS + ACTIONS
========================================================
Add authenticated endpoints:

1) GET /api/referrals/invite/:token
- Requires auth
- Finds referral by invite_token
- Validates:
  - not expired
  - referral status is pending
  - logged-in user’s company matches receiver_company_id
- Returns:
  - referral summary (fee type/value)
  - sender company name
  - job details needed for display (title, customer, address, scheduled time, scope/notes, price if allowed)

2) POST /api/referrals/invite/:token/accept
- Requires auth
- Same validation as above
- Accepts referral + transfers job ownership (reuse existing accept logic)
- Return updated job + referral

3) POST /api/referrals/invite/:token/decline
- Requires auth
- Mark declined
- Return referral

Important:
- Do NOT allow any company other than receiver_company_id to accept via token.
- Token is only a “locator”; authorization still enforced via logged-in company.
