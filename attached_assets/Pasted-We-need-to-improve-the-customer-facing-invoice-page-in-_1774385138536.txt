We need to improve the customer-facing invoice page in EcoLogic by showing more job/payment detail.

CURRENT ISSUE
Right now the invoice page is too bare.
It shows the company info, bill-to info, job name, subtotal/total, and pay button — but it does NOT clearly show the actual line items/materials/services that make up the invoice.

That is not enough.

GOAL
Make the invoice page more informative so the customer can clearly see what they are being charged for.

DESIRED ADDITIONS
Add a proper itemized breakdown section to the invoice page that shows the actual things included in the invoice.

This should include the line items/materials/services that were added to the estimate/job/invoice.

EXAMPLES OF WHAT SHOULD APPEAR
For each charged item, show useful details such as:
- item name
- optional description if one exists
- quantity
- unit price
- line total

If materials are included, those should appear too in a clean way.

If the invoice is based on both labor/service line items and materials, the customer should be able to understand exactly what was included.

CORE PRODUCT INTENT
The invoice should answer:
“What exactly am I paying for?”

not just:
“What is the total?”

WHAT TO BUILD

1. Add an itemized section to the invoice page
Create a clean invoice breakdown area below the header/customer/job info and above the totals/pay button area.

This section should list the invoice contents clearly.

2. Show the real billed entries
Use the actual invoice/job/estimate source-of-truth data so the displayed breakdown matches what the customer is being charged.

Do not fake or duplicate inconsistent data.

3. Include useful fields when available
Per entry, include:
- name
- description if present
- quantity
- unit price
- line total

Only show fields when they exist / make sense.
Keep it clean.

4. Handle materials too
If materials are part of the billed total, include them in the breakdown.
Do not leave them hidden.

5. Preserve clean invoice styling
This is customer-facing, so it should still look polished and readable.

Keep EcoLogic’s clean invoice aesthetic:
- simple
- professional
- easy to scan
- not cluttered

6. Make mobile/web layout still work
The invoice page should remain readable on mobile and web/browser invoice links.

7. Keep totals consistent
The itemized rows must reconcile with:
- subtotal
- total
- balance due

Do not create mismatched math between the line-item section and the totals section.

OPTIONAL IF DATA EXISTS
If there is already useful metadata available, also consider showing:
- service date
- notes
- grouped sections if needed

But only if it improves clarity and does not clutter the page.

IMPORTANT

- Do not break invoice totals

- Do not make the page overly dense
- Do not show internal-only data that customers should not see
- Keep it contractor-professional

with the totals below matching the breakdown.
