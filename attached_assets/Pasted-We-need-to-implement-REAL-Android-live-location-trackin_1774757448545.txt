We need to implement REAL Android live location tracking for EcoLogic technicians while clocked in.

Goal:
When a user clocks in on Android, EcoLogic should begin continuous live location tracking and keep updating location even if the app is backgrounded/minimized. When the user clocks out, tracking must stop immediately.

Important rules:
- Only track while the user is clocked in.
- Android must use the correct native permission + foreground service approach.
- Do NOT break iOS or web.
- Keep existing EcoLogic logic/UI intact unless needed for this feature.
- Add targeted logs only. No junk logs.

What Android must support:
1. Request precise location permission.
2. Request background location permission for Android if needed for tracking while app is not on screen.
3. Start a native Android foreground service of type location while clocked in.
4. Show the required persistent Android notification while tracking is active.
5. Send live location updates to the existing EcoLogic backend on an interval and/or distance threshold.
6. Stop service + watcher fully on clock-out, logout, forced clock-out, and timesheet/session end.
7. Restore tracking state correctly if the app resumes while the user is still clocked in.
8. Fail gracefully if permission is denied or location services are off.
