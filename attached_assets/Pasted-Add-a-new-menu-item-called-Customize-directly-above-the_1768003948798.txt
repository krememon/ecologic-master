2) Service Catalog tab
This is where company admins can create/edit reusable line items (services/materials) used in estimates.

UI SPEC (simple + clean)
- Top right button: “+ Add Service”
- Search input: “Search services…”
- List of service cards/rows showing:
  - Name
  - Description (optional)
  - Default price
  - Unit (each/hour/etc)
  - Category (optional)
  - Edit button (pencil) on each row
- Clicking Edit opens modal pre-filled
- Add Service opens same modal empty

MODAL FIELDS
- Name (required)
- Description (optional)
- Price (number, required)
- Unit (dropdown: each, hour, ft, job)
- Category (optional)
Buttons: Cancel / Save

DATA MODEL (company-scoped)
We need these stored per company so all users share them:
Table: service_catalog_items (or estimate_line_item_templates)
Columns:
- id
- company_id (required)
- name (required)
- description (nullable)
- default_price (numeric)
- unit (text)
- category (nullable)
- created_at, updated_at

API ROUTES
- GET /api/service-catalog
  -> returns items for req.user.companyId
- POST /api/service-catalog
  -> create item for req.user.companyId
- PATCH /api/service-catalog/:id
  -> update item (must belong to same company)
