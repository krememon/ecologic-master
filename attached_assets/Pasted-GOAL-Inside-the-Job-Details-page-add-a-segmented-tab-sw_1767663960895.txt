3) BACKEND: DB TABLES (JOB-SCOPED)
------------------------------------------------------------

Add tables (or equivalent) to DB:

TABLE: estimates
- id (uuid)
- company_id
- job_id (required)
- estimate_number (string, unique per company, ex: EST-000123)
- title (text)
- notes (text nullable)

- id (uuid)
- estimate_id
- name (text)
- quantity (decimal or int)

- sort_order (int)

------------------------------------------------------------
4) BACKEND: API ROUTES (RBAC + JOB ACCESS ENFORCED)
------------------------------------------------------------

Add these endpoints:

GET /api/jobs/:jobId/estimates
- Returns estimates for this job
- Enforce job access
- Enforce role access (no Technician)

POST /api/jobs/:jobId/estimates
- Creates a draft estimate for job
- Body includes title, notes, items
- Server calculates totals (do not trust client totals)
- Enforce job access + role access

GET /api/estimates/:estimateId
- Returns estimate + items
- Enforce access via estimate.job_id and user job access

PUT /api/estimates/:estimateId
- Update title/notes/items/status (draft updates)
- Server recalculates totals
- Enforce access

Estimate number generation:
- Create a company counter (company_counters table or similar):
  - estimate_counter int
- On create, increment atomically and format:
  - EST-000001, EST-000002, etc.
