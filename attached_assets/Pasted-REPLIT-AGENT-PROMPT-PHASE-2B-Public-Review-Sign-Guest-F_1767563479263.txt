REPLIT AGENT PROMPT — PHASE 2B (Public “Review & Sign” Guest Flow)

CONTEXT / SOURCE OF TRUTH
We are building EcoLogic. Keep UI clean + mobile-first. Do NOT break existing Docs, RBAC, or internal signature request creation/sending.
Current: email sends and links to /sign/:token but it redirects to EcoLogic login. We need a PUBLIC guest signing flow like DocuSign/Housecall Pro: customer clicks link → sees document → signs → done (NO EcoLogic account).

PHASE 2B GOAL
Implement a public signing experience for /sign/:token that:
1) does NOT require login
2) fetches signature request by token via public API
3) shows document preview + message
4) captures signature (draw pad) + submits
5) marks request as SIGNED + stores signature proof
6) shows confirmation screen
7) updates internal EcoLogic UI (signature request list) to show Signed status after refresh

IMPORTANT SECURITY RULES
- Public endpoints must NOT expose company internal data beyond what’s needed to sign.
- Token must be validated, and signing should be blocked if:
  - token not found
  - status != "sent"
  - already signed
  - expired (add expiresAt)
- Do NOT require auth for /sign/:token or /api/public/* routes.

BACKEND: PUBLIC API ROUTES (NO AUTH)
Create these routes in server/routes.ts (or equivalent). Place them under /api/public and ensure they bypass auth middleware.

A) GET /api/public/signature-requests/:token
Return safe payload:
{
  id,
  status,
  customerName,
  customerEmail (optional – ok to return),
  message,
  createdAt,
  sentAt,
  expiresAt,
  document: { id, name, category, mimeType, url }  // minimal for preview
  relatedJob: { id, title } (optional if already stored)
  company: { name, logoUrl? } (optional if easy; otherwise omit)
}

Validation:
- If not found -> 404 {message:"Invalid link"}
- If status != "sent" -> 400 {message:"This request is not available for signing"}
- If signedAt exists or status=="signed" -> 400 {message:"Already signed"}
- If expiresAt exists and now > expiresAt -> 410 {message:"Link expired"}

NOTE: Document URL must be directly accessible for preview. If existing storage URLs require auth, generate a signed/public URL for this endpoint only.

B) POST /api/public/signature-requests/:token/sign
Body:
{
  signatureDataUrl: "data:image/png;base64,...",   // from canvas
  signerName?: string
}

Validation:
- Same token checks as GET
- Must include signatureDataUrl
- signatureDataUrl must start with "data:image/png;base64,"
- Reject if too large (basic protection; e.g., > 2MB)

Storage:
- Convert dataUrl -> buffer and upload to storage (reuse existing upload helper if possible).
- Save signatureStorageKey/signatureUrl
- Set signedAt = now
- Set status = "signed"
- Set signedName = signerName || customerName

Response:
200 { ok:true }
