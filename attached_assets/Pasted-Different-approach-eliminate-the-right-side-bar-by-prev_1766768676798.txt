SYMPTOM
- A vertical “bar” appears on the right side in the phone preview.
- Prior scrollbar-hide attempt didn’t remove it, so it’s likely overflow/gutter caused by layout width > viewport, not a visible scrollbar style.

GOAL
- Chat page fits perfectly in the phone frame with no right-side bar.
- No horizontal scroll anywhere.
- Only the messages list scrolls vertically.
- Header and input always visible.

DO THIS NOW:

1) HARD LOCK CHAT ROOT
Find the outermost container for the Chat screen route (the page that renders the phone chat UI).
Replace its sizing rules with STRICT viewport lock:

- width: 100vw
- max-width: 100vw
- height: 100dvh
- max-height: 100dvh
- overflow: hidden
- overflow-x: hidden

Tailwind example:
<div className="w-[100vw] max-w-[100vw] h-dvh max-h-dvh overflow-hidden overflow-x-hidden flex flex-col bg-white">

If h-dvh not available, use inline style:
style={{ width:'100vw', maxWidth:'100vw', height:'100dvh', maxHeight:'100dvh', overflow:'hidden' }}

2) REMOVE ANY WRAPPER MAX-WIDTH / CENTERING
Search for any parent wrapper used across pages like:
- max-w-[...]
- container
- mx-auto
- padding on a root layout that squeezes the viewport
For the chat route ONLY, bypass those wrappers or override them with:
w-full max-w-none mx-0 px-0

This is often the real reason the phone preview shows a right gutter.

3) FORCE CHILDREN TO NOT OVERFLOW WIDTH
Add these rules to key containers:
- chat root: overflow-x-hidden
- messages scroller: overflow-y-auto overflow-x-hidden min-h-0 min-w-0
- message row wrappers: min-w-0
- bubble: max-w-[75%] break-words whitespace-pre-wrap

CRITICAL: add min-w-0 to any flex children (messages list / row) to prevent flex overflow.

Example:
Messages scroller:
<div className="flex-1 min-h-0 min-w-0 overflow-y-auto overflow-x-hidden">

Message row:
<div className="w-full min-w-0 flex ...">

Bubble:
<div className="max-w-[75%] break-words whitespace-pre-wrap">

4) REMOVE 1–4PX WIDTH OVERFLOW (COMMON)
If the bar is caused by 100vw + scrollbars, use this safer width:
width: 100%;
not 100vw, but only if the chat is inside a fixed-width frame already.

So implement a toggle:
- If your chat route is already inside a “phone frame” container in the preview, set chat root to:
  w-full max-w-full h-full max-h-full overflow-hidden
and ensure the phone frame itself is h-dvh/w-full.

Choose the correct one based on your layout:
- If chat is full-page route -> use 100vw/100dvh
- If chat is inside a phone preview component -> use 100%/100% and constrain the phone frame

5) GLOBAL CSS SAFETY NET (TARGETED)
Add a chat-specific class like .chat-viewport and apply:

.chat-viewport * { max-width: 100%; box-sizing: border-box; }

DO NOT apply this globally to the entire app; only the chat route container.
