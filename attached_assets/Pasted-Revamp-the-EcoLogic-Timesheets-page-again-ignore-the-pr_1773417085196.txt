Revamp the EcoLogic Timesheets page again — ignore the previous “employee list first” direction. We want the page to work more like the screenshots I attached.

NEW UX DIRECTION
The Timesheets page should display the DAYS OF THE WEEK first, and when the user taps/clicks a day, it should show the employees’ hours and entries for that selected day.

This should feel similar to the reference screenshots:
- top horizontal day selector
- selected day highlighted
- below that, a summary area
- below that, employee rows/cards for that selected day
- each employee can show total time for that day
- employee section can expand or already show their entries for that day

IMPORTANT
Do NOT break the current clock-in / clock-out functionality.
The backend time tracking logic already works:
- user clocks in
- time tracks
- user clocks out
- entry posts
Keep that behavior intact.
We are changing the UI/UX presentation, not the core tracking flow.

GOAL
Make Timesheets feel clean, useful, mobile-first, and easy to scan.

DESIRED PAGE STRUCTURE

1. HEADER
- Title: Timesheets
- Optional small subtext like: “Review daily employee hours”

2. TOP DAY-OF-WEEK SELECTOR
At the top of the page, show the current week as a horizontal selector like the screenshot:
- Sun / Mon / Tue / Wed / Thu / Fri / Sat
- each day shows weekday letter + date number
- selected day is highlighted with EcoLogic blue
- left/right arrows move to previous/next week
- selector updates all content below
- mobile-first, touch friendly, smooth scrolling if needed
- desktop should still look polished

3. DAILY SUMMARY CARD
Under the day selector, show a summary card for the selected day:
- Total Hours for company/team that day
- Optional:
  - total employees who clocked time that day
  - total entries that day
Only include stats that are easy to support with current data.

4. EMPLOYEE LIST FOR THE SELECTED DAY
For the chosen day, show all relevant employees for that date.
Each employee row/card should show:
- avatar or initials
- employee name
- total hours for that selected day
- maybe status if currently clocked in on today
- clean card layout, no ugly tables

5. EMPLOYEE DAY DETAIL
When clicking an employee row, show that employee’s entries for the selected day.
Use a card layout like the screenshot:
- Day header/date
- Each clock session entry:
  - label like Work
  - clock in time
  - clock out time
  - duration
- If there are multiple sessions in that same day, show them all
- Show daily total for that employee
- If the person is actively clocked in, clearly show Active and current running duration if safely supported

UX BEHAVIOR
- Default selected day should be today if it is in the visible week
- If viewing past/future week, default to first day or preserve last selected day index
- Tapping another day updates:
  - company total hours for that day
  - employee list for that day
  - each employee’s displayed totals for that day
- Tapping employee expands/collapses their detailed daily entries
- Must feel snappy and easy to understand on iPhone
