We need to integrate APPLE PAY into EcoLogic as a checkout payment method using Stripe.

GOAL
Allow customers to pay invoices with Apple Pay wherever Stripe supports it, while keeping existing card and other payment methods working.

IMPORTANT
This task is for Apple Pay as an online checkout wallet/payment option inside EcoLogic.
This is NOT the separate “Tap to Pay on iPhone” card-reader style in-person product unless the current architecture already explicitly uses that.

DO NOT BREAK
- existing Stripe card payments
- existing Klarna / other intended Stripe payment methods
- public/send-invoice checkout
- normal invoice checkout
- subcontract checkout if it shares Stripe flow
- successful payment confirmation
- QuickBooks sync after successful payment
- refunds

GOAL UX
When a compatible Apple device/browser/app context is available:
- customer should see an Apple Pay option
- tapping it should complete payment quickly
- payment should flow through Stripe and mark the invoice paid exactly like card payments do

When Apple Pay is NOT available:
- checkout should gracefully fall back to normal payment methods
- no broken buttons / dead UI

- normal invoice checkout

Implement Apple Pay using the best Stripe-supported approach for the current architecture.

- add Apple Pay in the cleanest way that works with the existing Stripe setup

Check and document what is required for Apple Pay to work:
