6) DB schema

push_device_tokens:

- created_at

Implement storage methods:

7) API routes
Create routes:
POST /api/push/register
- Auth required
- Body: { token, platform }
- Validate token exists; upsert for current user; return ok

POST /api/push/test
- Auth required (OWNER/ADMIN only)
- Body: { userId? } default current user
- Finds tokens for user and sends a test push via APNs sender (see #8)
- Return delivery result summary

8) APNs sender (token-based)
Create server module:
server/push/apns.ts
- Uses APNs HTTP/2 provider.
- Use environment secrets:
  APNS_TEAM_ID
  APNS_KEY_ID
  APNS_BUNDLE_ID (e.g., com.ecologic.app)
  APNS_AUTH_KEY_P8 (the full .p8 content OR base64)
- Implement sendPushToTokens(tokens, payload):
  payload example:
  {
    aps: {
      alert: { title: "EcoLogic", body: "You have been assigned a job." },
      sound: "default",
      badge: 1
    },
    data: { type: "job_assigned", jobId: "..." }
  }
- Use production vs sandbox based on NODE_ENV or a secret APNS_USE_SANDBOX=true for dev builds.
- Handle errors: Unregistered token -> deactivate it.

NOTE:
- Do NOT invent the APNs library; pick a stable Node APNs solution (e.g., “apns2” or direct HTTP/2).
- Keep code small and well-logged behind a [push] prefix.
