API ROUTES
Create new backend endpoints:
1) POST /api/campaigns/preview
Input: { clientIds: number[], channel: 'email'|'sms'|'both' }
Return: counts + arrays of eligible clientIds per channel, plus excluded summary reasons.
This endpoint must enforce RBAC and company_id scoping.

2) POST /api/campaigns/send
Input: { clientIds: number[], channel, subject?, emailBody?, smsBody? }
Behavior:
- Validate RBAC
- Create campaign row
- For each eligible recipient, create campaign_recipients row and attempt send
- Email: integrate with provider (see below) or if not configured yet, implement a “dev mode” transport:
   - If EMAIL_PROVIDER_KEY missing, DO NOT silently succeed; mark as failed with clear error_message
- SMS: integrate with provider (Twilio) or similar dev-mode rules:
   - If TWILIO creds missing, mark as failed
- Return: campaignId + results summary

3) POST /api/webhooks/sms (Twilio inbound)
- Handle STOP/UNSUBSCRIBE keywords
- Find client by phone and set sms_opt_in=false and sms_unsubscribed_at=now
- Return 200

4) GET /api/campaigns (optional for later): list last 20 campaigns for company; can stub but not required.
