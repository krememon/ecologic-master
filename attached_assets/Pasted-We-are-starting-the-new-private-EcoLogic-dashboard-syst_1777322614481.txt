PART 1 — Domain routing

Add support for hostname-based routing.

If the user visits:

staging-dashboard.ecologicc.com

they should see the private dashboard app/shell.

If the user visits:

staging.ecologicc.com

they should see the normal EcoLogic customer app.

Do not replace the customer app.

The dashboard should be a separate private internal experience.

PART 2 — Dashboard access control

The dashboard must require login.

Only approved owner/admin users can access it.

Normal EcoLogic users should be blocked, even if they have an account.

Use the existing auth system if possible.

Add a server-side and client-side access check.

Allowed roles:
- owner
- admin

Also support an env variable for hardcoded dashboard admin emails if useful:

DASHBOARD_ADMIN_EMAILS=ppellegrino@ecologicc.com

Rules:
- If not logged in, redirect to login.
- If logged in but not approved, show “Access denied.”
- Never expose dashboard data to normal users.
