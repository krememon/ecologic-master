Goal: Implement role-based access control (RBAC) for EcoLogic and add a Role selector to the registration page. Enforce permissions in the API and UI (hide/disable where needed). Keep existing auth/session intact.

Roles (exact)

OWNER — full access to everything.

SUPERVISOR — same permissions as OWNER (title difference only).

TECHNICIAN (Field Worker) — can view assigned jobs, update job status, upload photos, mark tasks complete.

DISPATCHER (Scheduler) — manages calendar, routes, appointment assignments.

ESTIMATOR (Sales Rep) — creates estimates, sends quotes, converts leads into jobs.

Note: OWNER and SUPERVISOR are equivalent in capability.

export function can(userRole: keyof typeof ROLE_PERMS, p: Perm) {

}

3) API guards (server)

Files: API route handlers (jobs, schedule, estimates, org/users).

Create middleware requirePerm(perms: Perm | Perm[]) that:

reads session user,

checks can(user.role, perm),

returns 403 if unauthorized.

Apply guards:

Jobs

GET /jobs → if user has jobs.view.all return all; else if jobs.view.assigned return only jobs assigned to user.id.

POST /jobs → jobs.create

PATCH /jobs/:id → jobs.edit (but allow TECHNICIAN to PATCH limited fields if jobs.status.update or jobs.tasks.complete).

POST /jobs/:id/photos → jobs.photos.upload.

Schedule

GET /schedule → schedule.view

POST/PATCH/DELETE schedule items → schedule.manage.

Routes → routes.manage

Estimates/Leads

POST /estimates → estimates.create

POST /estimates/:id/send → estimates.send

POST /leads/:id/convert → leads.convert

Org/Settings/Users → org.manage

Acceptance: Unauthorized calls return 403; authorized paths work.
