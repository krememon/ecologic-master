BUT ❌ the route returned 500 because of:

at server/routes.ts:13530

So the backend is doing the work, then crashes while building the response (or post-processing). This must be fixed because it:

Fix the crash by defining or removing newBalanceDueCents, and ensure /api/payments/manual returns 200 with the updated computed invoice state.

PRIMARY FIX

Search for: newBalanceDueCents

  ok: true,
  payment,
  balanceDueCents: newBalanceDueCents,
  ...
})

But that variable was renamed during the Stripe refactor (maybe to balanceDueCents or owedCents) and this route wasn’t updated.

2) Replace newBalanceDueCents with the correct value:
- If you already recompute invoice totals, use:
  const { balanceDueCents } = recomputeResult
OR
  const owedCents = recomputed.balanceDueCents

If you don’t have it in scope, compute it safely:
- const newBalanceDueCents = Math.max(0, invoice.total_cents - sumPaidCents)

But BEST: use the SAME recompute helper you already call (since it’s the source of truth).

3) Ensure route returns consistent response shape used by the frontend
Return:
{
  ok: true,
  paymentId,
  invoiceId,
  paidAmountCents,
  paidAmountTotalCents (sum),
